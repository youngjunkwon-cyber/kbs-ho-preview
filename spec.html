디자인 컨펌본 기반 섹션·구성요소별 기능 정의 · 노출 조건 · 데이터 소스 · 상호작용 정의
공통

공통 정책

화면 구조

  • 홈탭 롱스크롤 구조 (AS-IS의 Info/Products/Gallery/Reviews 4탭 폐지)
  • 별도 전체보기 페이지 없음 — 앵커 기반 홈탭 내 통합. "전체보기" 버튼은 해당 섹션 전체 펼침으로 동작
  • 상단 Header는 스크롤 위치에 따라 Default / Scrolled(축소) 2상태 전환

로그인 정책

  • 스토어 상세 페이지 자체는 비로그인 접근 가능
  • Sticky CTA의 채팅 진입도 비로그인 게스트 가능 (KBS 채팅 정책)
  • 예약·리뷰 작성은 로그인 필수
  • 저장·공유 등 기타 액션의 로그인 분기는 각 액션별로 별도 검토

데이터 소스 구분

  • 입점사 입력: 이미지·소개·시술·SNS·FAQ 등 대부분의 콘텐츠
  • 시스템 자동: 평점·리뷰수·영업상태(운영시간 기반)·거리
  • KBS·입점사 작성: Testimonials 대표 후기는 KBS 또는 입점사가 직접 작성·지정
  • KBS 템플릿: 예약/취소 정책·FAQ 기본 템플릿 제공 (FAQ는 입점사 커스텀 가능)
