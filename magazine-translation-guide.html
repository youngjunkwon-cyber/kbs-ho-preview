1-1 매거진 목록 화면 구성
1-1-1
조회 방법 및 버튼별 역할
목록 화면 전체 구조
SCREENSHOT · 매거진 목록 화면
매거진 목록 화면에서 글을 조회하고, 신규 작성·복제·삭제·상태 변경을 수행합니다. 행별로 노출 통계를 한눈에 확인할 수 있습니다.
  • ●필터: 상태 / 언어 / 작성일 + 검색어로 조회 후 [검색] 클릭
  • ●[매거진 작성] (우측 상단): 새 매거진 작성 화면 진입
  • ●통계 컬럼: 노출수 / 내부진입수 / 직접유입 / 대표블록수 / 상품 갯수 / 스토어 갯수
  • ●관리 컬럼: 행별 [복제] · [삭제] 버튼
  • ●상태 드롭다운: 행별로 게시 / 비게시 / 검토필요 변경
💡 모든 컨텐츠는 개별 관리되며, 복제하여 게시한 경우도 각 언어 별로 컨텐츠가 관리됩니다.
1-2 작성 화면 구성
1-2-1
필드별 설명
작성 화면 상단 필드 영역
SCREENSHOT · 작성 화면 필드 영역
매거진 작성 화면은 상단 기본 정보 영역과 하단 본문 작성 및 미리보기 영역으로 구성됩니다. 기본 정보 필드의 입력 규칙은 아래와 같습니다.
  • 내부 관리용 제목
    관리자만 보는 식별용 제목. FO 노출 안 됨. 운영팀이 글을 구분하기 위한 라벨로 사용됩니다.
  • 게시 언어
    FO에 어느 언어로 노출할 것인지 선택. 영어 · 일본어 · 중국어 · 중국어(번체) · 인도네시아 중 1개. 자동번역 · 상품/스토어 선택 시 기본 언어 설정으로도 사용됩니다.
  • 노출 제목
    FO에 실제로 보이는 제목. 언어 탭별로 별도 입력. 자동번역 시 함께 번역되어 채워집니다.
  • Slug
    URL 경로용 키워드. 노출 제목 키워드로부터 자동 생성되며 직접 수정 가능. [중복검사] 버튼으로 사용 가능 여부 확인. 중복 시 저장 불가.
  • 대표 태그
    [대표 태그 선택] 클릭 → 모달에서 1개 선택. 글의 메인 카테고리로 사용.
  • 기타 태그
    [기타 태그 추가] 클릭 → 모달에서 다중 선택. 보조 분류·검색 정확도 향상에 사용.
  • 미리보기
    우측 패널에서 실시간 FO 노출 형태 확인. 375px / 480px 토글로 디바이스 폭 전환. 에디터 입력 시 자동 갱신.
  • 대표 이미지
    우측 영역의 [이미지 업로드]. 미리보기 카드 썸네일·소셜 공유 시 사용되는 메인 이미지.
1-2-2
에디터 콘텐츠 추가 — 모달 4종
태그 / 상품 / 스토어 / 이미지 업로드
에디터 본문에 콘텐츠를 추가하는 4가지 모달입니다. 각 모달은 에디터 툴바의 해당 버튼 클릭 시 활성화됩니다.
태그 선택 모달
대표 태그 / 기타 태그 TAG
사전 등록된 태그 풀에서 선택. 5개 언어 매핑이 사전에 잡혀있어 언어 탭 전환 시 자동 반영. 대표는 1개, 기타는 다중.
상품 추가 모달
상품 추가 PRODUCT
상품 검색 후 본문에 카드 형태로 삽입. 자동번역 대상에서 제외됩니다.
스토어 추가 모달
스토어 추가 STORE
스토어 검색 후 본문에 카드 형태로 삽입. 자동번역 대상에서 제외됩니다.
이미지 업로드 모달
이미지 업로드 IMAGE
파일 드래그 앤 드롭 또는 선택 업로드. 업로드 후 본문에 자동 삽입.
📌 상품 / 스토어 위젯은 자동번역 대상에 포함되지 않습니다. 각 위젯은 자체 다국어 데이터를 사용하기 때문입니다.
1-2-3
자동번역 기능
에디터 툴바 → [자동번역]
SCREENSHOT · 자동번역 실행 / 팝업
에디터 본문 작성 후 다른 언어로 번역할 때 사용합니다. 현재 활성 탭의 본문을 선택한 언어로 자동번역하여 덮어씁니다.
  • 1번역할 언어 탭으로 전환
  • 2에디터 툴바 → [자동번역] 클릭
  • 3확인 팝업에서 [자동번역] 버튼 클릭
  • 4번역 진행 → 완료 후 본문 자동 갱신
⚠️ 번역 중에는 페이지를 이탈하거나 다른 탭을 클릭하지 마세요. 번역이 중단될 수 있습니다.
💡 노출 제목도 함께 자동번역됩니다. 어색하면 직접 수정 가능합니다.
1-2-4
저장
우측 상단 [저장] 버튼
SCREENSHOT · 저장 버튼
모든 언어의 본문 작성·번역이 끝나면 우측 상단 [저장] 버튼으로 한 번에 저장합니다.
  • ●화면 우측 상단 [저장] 클릭
  • ●저장 후 게시 상태는 비게시로 진입
  • ●목록으로 돌아가려면 [목록으로]
💡 언어별로 따로 저장하지 않아도 됩니다. 모든 언어 탭을 채운 뒤 한 번에 저장하면 됩니다.

PART 02
복제
기존 글을 복제하여 다국어 번역본을 빠르게 만드는 절차입니다. 작성 절차 대비 입력 단계가 단축됩니다.
2-1
복제 버튼 클릭
매거진 목록 → 행별 [복제] 버튼
SCREENSHOT · 복제 버튼
매거진 목록에서 복제하려는 글의 [복제] 버튼을 클릭합니다. 모든 콘텐츠(본문·필드·태그·상품·스토어·이미지)가 그대로 복사됩니다.
  • ●목록에서 원본 글 행 찾기
  • ●행의 [복제] 버튼 클릭
  • ●목록 상단에 복사본 생성 확인
  • ●복사본 클릭하여 작성 화면 진입
📌 복제된 글은 비게시 상태로 생성됩니다. 검수·수정 후 게시 상태로 변경 필요.
2-2
자동번역 동작
언어 탭별 자동번역 실행
SCREENSHOT · 복제 후 자동번역
복제된 글의 모든 언어 탭은 원본과 동일한 내용으로 채워져 있습니다. 각 언어 탭에서 자동번역을 실행하여 해당 언어로 변환합니다.
  • 1JA 탭 → [자동번역] → 일본어로 변환
  • 2ZH 탭 → [자동번역] → 중국어 간체로 변환
  • 3ZH-TW 탭 → [자동번역] → 중국어 번체로 변환
  • 4ID 탭 → [자동번역] → 인도네시아어로 변환
💡 EN(영어)은 원본 언어이므로 자동번역 없이 그대로 사용합니다. 필요시 직접 수정만 합니다.
2-3
노출 제목 / slug 변경 유의
중복 방지 — 반드시 변경 필요
SCREENSHOT · slug 중복검사
복제 시 노출 제목과 slug는 원본과 동일하게 들어옵니다. 반드시 변경해야 저장이 가능합니다.
  • !노출 제목: 자동번역 후에도 어색하면 직접 수정
  • !slug: 원본과 중복됨 → 새로운 값 입력 필수
  • ![중복검사] 버튼으로 사용 가능 여부 확인
  • !중복 시 저장 불가 — 다른 값으로 변경
⚠️ slug를 변경하지 않고 저장 시도 시 오류 메시지가 표시됩니다. 반드시 중복검사 통과 후 저장하세요.
2-4
저장
우측 상단 [저장] 버튼
SCREENSHOT · 저장
모든 언어 탭의 자동번역과 노출 제목·slug 검수가 끝나면 [저장]합니다.
  • ●화면 우측 상단 [저장] 클릭
  • ●저장 후 목록에서 게시 상태 변경 (게시 / 비게시 / 검토필요)
  • ●목록으로 돌아가려면 [목록으로]
💡 게시 상태는 목록 화면에서 행별 드롭다운으로 변경합니다. 게시 선택 시 FO에 즉시 노출됩니다.
